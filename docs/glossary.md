# 附录 B｜术语表：读教程时随手查

按主题分组。每个词条给一句"人话"解释，再补一句它为什么重要。教程正文第一次出现这些词时，都可以回到这里对照。

## 模型与调用

- **LLM（大语言模型）**：根据已有文本预测下一段文本的概率模型。它没有内置数据库，也不会主动"做事"，一切能力都来自你在输入里提供的信息和你允许它触发的工具。
- **Token**：模型处理文本的最小单位，大致相当于一个词或半个汉字到几个汉字不等。计费和上下文长度都按 Token 计算，所以"少塞无关内容"直接省钱。
- **上下文窗口（Context Window）**：模型一次调用能"看见"的最大 Token 数。像一张工作台而不是硬盘——放不下的东西模型不知道，堆太满关键内容会被淹没。
- **消息角色（Role）**：`system`（开发者定的规则）、`user`（用户输入）、`assistant`（模型输出）、`tool`（工具执行结果）。模型对不同角色的"信任程度"不同，但这不是安全边界。
- **温度 / 采样（Temperature / Sampling）**：控制输出随机性的参数。分类、路由这类要稳定的任务用低随机性；创意写作可以调高。
- **流式输出（Streaming）**：边生成边返回，改善用户等待体验，但不会减少总耗时。
- **幻觉（Hallucination）**：模型自信地编造不存在的事实。缓解手段是提供依据（RAG）、要求引用、允许"不知道"，而不是在 Prompt 里写"不要幻觉"。

## Prompt 与输出

- **Prompt / System Prompt**：给模型的输入文本 / 其中由开发者预设的规则部分。注意：Prompt 是"恳请"，不是权限系统。
- **Structured Output（结构化输出）**：要求模型按 JSON Schema 返回结果，程序可以直接解析成对象。回答"最终答案长什么样"。
- **Tool Calling / Function Calling（工具调用）**：模型返回"我想调用某工具 + 参数"的结构化建议，由你的程序决定是否执行。回答"中途需要什么能力"。
- **JSON Schema**：描述 JSON 结构的标准格式。工具参数和结构化输出都靠它约定。
- **Pydantic**：Python 数据校验库。一个 Pydantic 模型同时是类型注解、运行时校验器和 JSON Schema 来源，是本教程里"合同"的载体。
- **Few-shot（少样本示例）**：在 Prompt 里给几个输入输出示例，帮模型理解格式和边界情况。

## Agent 核心

- **Agent**：围绕目标运行的程序：让模型在受控范围内读上下文、选工具、观察结果、更新状态，直到完成或触发停止条件。
- **Agent Loop**：上述过程的循环骨架：模型判断 → 程序执行 → 结果回传 → 再判断。
- **ReAct**：Reason + Act 的缩写，"判断—行动—观察"交替进行的模式，是大多数工具调用 Agent 的底层套路。
- **Observation（观察结果）**：工具执行后回传给模型的结果，包括成功数据和结构化的错误。
- **Tool（工具）**：一个有名称、描述、参数 schema、稳定输出和错误语义的能力，例如"查订单"。
- **Allowlist（白名单）**：模型只能调用登记过的工具，防止它"发明"调用任意函数。
- **停止条件（Stop Policy）**：最大步数、超时、预算上限等强制刹车，防止无限循环和成本失控。
- **Workflow（工作流）**：由开发者预先定义步骤和分支的确定性流程；与 Agent 的区别是"谁决定下一步"。
- **HITL（Human-in-the-loop，人工介入）**：高风险动作前暂停，等人批准、修改或拒绝后再继续。

## LangChain / LangGraph

- **LangChain**：高层 Agent 框架，统一模型、消息、工具接口，`create_agent` 一行创建标准工具调用 Agent。
- **LangGraph**：底层编排运行时，用 State（状态）+ Node（节点）+ Edge（边）显式表达分支、循环、持久化和审批。LangChain 的 Agent 就跑在它上面。
- **State**：图运行过程中的"当前世界快照"，一个类型化的字典。
- **Node（节点）**：读取状态、做一件事、返回状态增量的函数。
- **Edge（边）**：定义节点之间的转移；条件边根据状态决定下一步去哪。
- **Reducer**：多个节点更新同一个状态字段时的合并规则（覆盖、追加、求和……）。
- **Checkpoint / Checkpointer**：每步执行后保存的状态快照 / 负责保存它的组件。有了它才能暂停、恢复和回放。
- **Thread**：一条对话/任务的持续标识。同一 `thread_id` 共享 checkpoint 历史，不同 thread 彼此隔离。
- **Interrupt**：LangGraph 中让图暂停等待外部输入（如人工审批）的机制，配合 `Command(resume=...)` 恢复。

## RAG 与记忆

- **RAG（检索增强生成）**：先从知识库检索相关材料，再让模型基于这些材料回答。解决"模型不知道你的私有/最新数据"。
- **切块（Chunking）**：把长文档切成适合检索的小段，每段保留出处。
- **Embedding（向量嵌入）**：把文本转成数字向量，语义相近的文本向量距离近，是语义检索的基础。
- **向量数据库（Vector Store）**：存储向量并支持"找最相似"查询的数据库。
- **BM25 / 关键词检索**：基于词频的经典检索算法，对专有名词、编号、错误码效果好。
- **Hybrid Search（混合检索）**：关键词 + 向量两路召回再融合，通常比单一路径稳。
- **重排（Rerank）**：对初步召回的候选用更精细的模型重新排序，把最相关的排到前面。
- **Recall@K**：正确文档出现在前 K 条召回结果中的比例，检索层最常用的指标。
- **短期记忆 / 长期记忆**：一个会话内延续的内容（checkpoint）/ 跨会话保留的用户偏好与事实（需要治理：来源、过期、删除机制）。

## 协议与多 Agent

- **MCP（Model Context Protocol）**：让 AI 应用以统一协议发现并调用外部工具、资源和提示的开放标准，好比"Agent 世界的 USB 接口"。
- **MCP Host / Client / Server**：使用能力的 AI 应用 / 它内部维护连接的组件 / 提供能力的服务。
- **Router / Supervisor / Handoff**：多 Agent 的三种协作模式：入口分类转交、主管委派汇总、当前 Agent 移交控制权。

## 工程与生产

- **Guardrail（护栏）**：对输入、输出、工具调用和运行轨迹的确定性检查，在模型"想做坏事"或"被骗"时兜底。
- **Prompt Injection（提示注入）**：攻击者把恶意指令藏进用户输入或外部内容（网页、邮件、文档），诱导模型越权行动。核心防御：外部内容永远当数据，不当指令。
- **幂等（Idempotency）**：同一操作执行多次与执行一次效果相同。写操作配幂等键，重试才不会重复退款、重复发货。
- **重试与退避（Retry / Backoff）**：只对暂时性错误重试，且间隔按指数增长并加随机抖动，避免"重试风暴"。
- **超时（Timeout）/ 熔断（Circuit Breaker）**：单步等待上限 / 依赖持续故障时主动停止调用，防止拖垮整个系统。
- **Trace / Span**：一次请求经过的完整路径记录 / 其中的单个步骤。排查"这次到底发生了什么"的第一手证据。
- **可观测性（Observability）**：Trace + 日志 + 指标的总称，让系统的内部状态可以被外部看见。
- **离线评测（Offline Evaluation）**：在固定数据集上批量运行并打分，回答"改完之后真的更好吗"。
- **LLM-as-judge**：用一个模型给另一个模型的输出打分，适合"有帮助吗"这类模糊维度，需要人工校准。
- **SSE（Server-Sent Events）**：服务器向浏览器单向推送事件的 HTTP 机制，常用于流式返回 Agent 进度。
- **金丝雀发布（Canary）**：先放小部分流量到新版本，指标正常再全量。
- **Kill Switch**：一键禁用某个模型或危险工具的开关，出事时的最后手段。
