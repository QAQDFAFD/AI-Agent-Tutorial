# 00｜AI Agent 全景地图：先看森林，再走进树林

如果把普通聊天模型比作一位“坐在房间里回答问题的聪明顾问”，Agent 就是给这位顾问配上电话、数据库、工具箱、工作笔记和办事流程。模型仍然负责判断，但真正让它能完成任务的是周围那套系统。

> 开始之前：还没把项目跑起来的话，先花 15 分钟按[附录 A：环境准备](setup.md)搭好环境；阅读中遇到陌生名词，随时查[附录 B：术语表](glossary.md)；想追溯这些设计思想的出处，见[附录 C：精读文选](reading-list.md)。

## 0.1 一个实用定义

本教程把 Agent 定义为：

> 一个围绕目标运行的程序。它让模型在受控范围内读取上下文、选择工具、观察结果、更新状态，直到完成任务或触发停止条件。

可以简化成一个循环：

```text
目标 → 判断 → 行动 → 观察 → 更新状态 → 再判断 → 完成/停止
```

“受控范围”和“停止条件”非常重要。没有它们的 Agent，不是更聪明，只是更容易无限循环、误调用工具或产生不可预测的副作用。

## 0.2 能力地图

![学习路线](../assets/diagrams/learning-roadmap.svg)

### A. Python 与软件工程基础

至少需要熟悉：

- 函数、类、类型注解、`dataclass`、异常处理与上下文管理器；
- `dict/list` 与 JSON，HTTP 请求，环境变量；
- `async/await`、并发限制、超时与取消；
- `pytest`、日志、Git、虚拟环境和依赖管理；
- Pydantic 模型与校验。

不要求先成为 Python 专家，但如果看到 `TypedDict`、泛型、异步迭代器就完全陌生，建议边学边补。Agent 的很多线上故障，本质是普通后端问题，不是“大模型玄学”。

### B. LLM 基础

要理解而不是背诵：

- 消息角色、上下文窗口、Token、采样与确定性；
- 模型没有自动访问你公司的数据库，也不知道“刚才工具执行成功没有”；
- Prompt 是运行时输入的一部分，不是可靠的权限系统；
- Structured Output 负责“输出长什么样”，Tool Calling 负责“要调用什么能力”；
- 流式输出改善体感，但不会自动降低总耗时；
- 推理模型也需要边界、验证和评测。

### C. Agent 核心

- **Agent Loop**：模型提出行动，程序校验并执行，结果送回模型；
- **Tool**：有清晰名称、描述、输入 schema、输出和错误语义的能力；
- **State**：本次任务进行到了哪里；
- **Memory**：需要跨步骤或跨会话保留的内容；
- **RAG**：按需找资料，再把有限且相关的上下文交给模型；
- **Workflow**：把确定性步骤写成图，不把所有决策都扔给模型；
- **HITL**：高风险动作前暂停，让人批准、修改或拒绝；
- **Multi-Agent**：多个专长或上下文边界明确的 Agent 协作；
- **MCP**：用统一协议发现并调用外部工具/资源。

### D. 生产工程

- 输入/输出/工具护栏、最小权限、审计与密钥管理；
- 重试、超时、熔断、幂等、缓存与限流；
- Checkpoint、任务队列、状态数据库；
- Trace、指标、日志、反馈；
- 离线数据集、确定性规则、LLM-as-judge、人工评审；
- 延迟、Token、模型路由和成本预算。

## 0.3 框架到底学哪些

### 主线：LangChain + LangGraph

官方当前把两者定位得很清楚：LangChain 是高层 Agent 框架，LangGraph 是底层编排运行时。LangChain 的 `create_agent` 本身运行在 LangGraph 上。学习顺序应当是：

1. 先手写一次循环，理解工具调用协议；
2. 用 LangChain 快速搭建常见 ReAct/工具调用 Agent；
3. 用 LangGraph 显式表达复杂状态、分支、循环、持久化与人工审批。

### 应该了解的替代方案

- **OpenAI Agents SDK**：如果主要使用 OpenAI，且希望开箱即用 handoff、guardrail、session 和 tracing，它很合适。
- **LlamaIndex**：数据连接、索引、检索和 Agentic RAG 是强项。
- **PydanticAI**：偏 Python 类型化体验，适合看重依赖注入与 Pydantic 集成的团队。
- **AutoGen / CrewAI**：提供多 Agent 交互模式；使用前先证明多 Agent 确实比单图更好。
- **Temporal / Inngest 等**：当工作流需要跨小时或跨天、强可靠定时与外部事件时，通用持久化工作流引擎也值得考虑。

不要同时深学所有框架。Agent 的共同原理比框架表面 API 更耐久。

## 0.4 推荐学习方式

每章都按下面四步走：

1. **读概念**：知道它解决什么故障，而不只是能写什么代码；
2. **跑 Demo**：先用默认输入跑通；
3. **制造故障**：传错参数、让工具超时、删掉停止条件；
4. **做评测**：把这次故障写成测试样例，防止以后复发。

### 毕业标准

当你可以独立完成下面这些事，就算掌握了 Agent 开发主干：

- 判断一个需求到底需要普通 LLM、RAG、Workflow 还是 Agent；
- 定义类型安全且权限收敛的工具；
- 画出状态、节点、路由和失败恢复路径；
- 给写操作加入人工审批和幂等键；
- 构造至少 20 条覆盖正常、边界、对抗输入的评测集；
- 从一次 Trace 中定位错误发生在模型、检索、工具还是编排；
- 把 Agent 包装成有认证、超时、限流和持久化的 API 服务。
